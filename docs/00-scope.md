\# CatalogHub (Product Catalog / Mini Shop System) – V1 Scope

Full-stack product catalog and user management application with modern Angular frontend and Spring Boot backend for learning, portfolio building, and interview preparation.





\## Project Goals

\- Build a clean, modern \*\*Angular + Spring Boot\*\* full‑stack application.

\- Demonstrate strong understanding of:

&#x20; - UI/UX flows

&#x20; - role-based access control

&#x20; - authentication + authorization (JWT)

&#x20; - REST API design (OpenAPI-driven)

&#x20; - database design \& ORM mapping

&#x20; - admin + customer-facing features

\- Keep V1 intentionally \*\*small, focused, and shippable\*\*.





\## In Scope (V1 Features)



\### Public (Guest)

\- Homepage

\- Product listing

\- Product details page

\- Register

\- Login



\### Customer

\- View profile

\- Manage delivery addresses

\- Manage billing addresses



\### Admin

\- View all products

\- Add product

\- Edit product

\- Delete product





\## Out of Scope (V1)

These may come in V2, but \*\*not now\*\*:

\- Cart

\- Orders

\- Payments

\- Coupons

\- Reviews

\- Notifications

\- Inventory management





\## User Roles

\- Guest

\- Customer

\- Admin





\## High-Level Tech Decisions

\- \*\*Frontend:\*\* Angular (standalone components, route-based feature modules, Future: signals)

\- \*\*Backend:\*\* Spring Boot (REST APIs, JPA/Hibernate)

\- \*\*Auth:\*\* JWT-based authentication

\- \*\*DB:\*\* MySQL / PostgreSQL

\- \*\*API Contract:\*\* OpenAPI (YAML)

\- \*\*Documentation:\*\* Markdown + draw.io + Mermaid

\- \*\*Testing:\*\* Postman collections





\## Success Criteria for V1

\- All V1 features implemented end-to-end.

\- Clean, structured codebase.

\- All APIs documented via OpenAPI.

\- Postman collection ready.

\- Basic UI working on desktop.

\- Ready to demo for interviews.
